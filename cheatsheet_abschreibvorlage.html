1. Scheduling

  1. Tasks notieren: A Ankunft, C Rechenzeit, D Deadline.
  2. Bei D=A+x: absolute Deadline = A+x.
  3. Nur bei Ereignissen neu entscheiden: Ankunft, CPU-Ende, Quantum, Deadline.
  4. Ready Queue pruefen: bereit != blockiert.
  5. Am Ende: Summe CPU-Balken je Task = C.
FCFSerst bereit, erst dran.
SJFkuerzester Job zuerst.
SRTkuerzeste Restzeit, verdraengend.
RRQuantum, dann hinten anstellen.
EDFkleinste absolute Deadline gewinnt.
RMSkleinste Periode = hoechste Prio.
Fallen: EDF absolute Deadline! RMS nicht nach Rechenzeit sortieren.

2. RMS / Echtzeit

U = sum(C_i / P_i)
U > 1 => auf 1 CPU unmoeglich.
U <= n(2^(1/n)-1) => RMS garantiert.

Werte: n=2: 0,828 | n=3: 0,779 | n->inf: 0,693.

Grenze ist hinreichend, nicht notwendig.

Antwortzeit-Test

R_i = C_i + sum(ceil(R_i/P_j)*C_j)

Nur hoehere Prioritaeten j summieren. Starte mit R=C_i, iteriere bis stabil. OK wenn R_i <= D_i.

3. Prozesse / Threads

ProzessProgramm in Ausfuehrung + eigener Adressraum + Ressourcen + PCB.
Threadeigener Stack/Register, teilt Adressraum im Prozess.
Readylauffaehig, wartet auf CPU.
Blockedwartet auf Ereignis/E-A/Semaphore.
Switchalten Zustand sichern, neuen laden, Dispatcher startet.
Verdraengung != Blockieren. Verdraengung = CPU wird entzogen.

4. Paging / Adressen

PageSize = 2^k => Offsetbits = k.
Seitennr-Bits = Adressbits - Offsetbits.
#Pages = 2^(Seitennr-Bits).
#Frames = phys. Speicher / PageSize.
Seitentabelle = Eintraege * Bytes/Eintrag.
4 KiB = 2^12 => 12 Offsetbits = letzte 3 Hex-Ziffern.

Beispiel: 0x4B30A568 => Page 4B30A, Offset 568.

Physische Adresse

  1. Virtuelle Adresse splitten: Page + Offset.
  2. Page in Seitentabelle nachschlagen => Frame.
  3. Physische Adresse = Frame + Offset.

Array ueber Pages

Pages = floor((Offset + Laenge - 1)/PageSize)+1

5. TLB / Page Fault

EAT = h*T_hit + (1-h)*T_miss

TLB-Hit: TLB + Datenzugriff. Miss: TLB + Seitentabelle + Datenzugriff.

10 ns TLB, 100 ns RAM, h=90%: Hit=110, Miss=210, EAT=120 ns.

Page Fault Ablauf

  1. Present-Bit 0 => Trap.
  2. Prozess blockieren.
  3. Freien Frame suchen / Opfer waehlen.
  4. Dirty Opferseite zurueckschreiben.
  5. Seite laden, Tabelle/TLB aktualisieren.
  6. Instruktion spaeter erneut ausfuehren.

Thrashing: Working Sets passen nicht in Frames => viele Page Faults. Fix: Multiprogramminggrad senken.

6. Replacement / Cache

FIFOersetzt Seite, die am laengsten im Speicher ist.
LRUersetzt Seite, die am laengsten nicht benutzt wurde.
ClockUse=1 -> 0 und weiter; Use=0 ersetzen.
OPTersetzt Seite, die am spaetesten wieder gebraucht wird.
  1. Referenzen links nach rechts.
  2. Bei Hit: LRU/Use-Bit aktualisieren.
  3. Bei Fault: laden oder Opfer nach Regel.
  4. Nach jedem Schritt alle Frames notieren.

Cache

Misses = ceil(Bytes / Blockgroesse)
HitRate = Hits / Zugriffe

200 B, Block 64 B, Cold Cache => 4 Misses, 196 Hits, 98%.

7. Synchronisation

Race Condition: Ergebnis haengt vom zeitlichen Ablauf nebenlaeufiger Zugriffe ab.

Kritischer Abschnitt: Code mit gemeinsamer Ressource, exklusiv schuetzen.

Mutexexklusiver Eintritt, initial 1.
SemaphoreZaehler mit atomarem wait/signal.
MonitorDaten + Prozeduren gekapselt, automatisch exklusiv.
Cond. Var.wait gibt Monitor frei, signal weckt.
Mutex: wait(S); krit. Abschnitt; signal(S)

Initialwert: 1 = freier Mutex, 0 = Reihenfolge/Event, N = Ressourcenpool.

8. Semaphor-Code

  1. Initialwerte an Code schreiben.
  2. Nach jedem wait: "haelt S".
  3. Bei wait auf belegtes S: "wartet auf S".
  4. Bei signal: S wird freigegeben.
  5. Deadlock suchen: P1 haelt S1 wartet S2, P2 haelt S2 wartet S1.
  6. Mehrere Locks immer gleiche Reihenfolge nehmen.
Klassiker: P1 wait(S1),wait(S2); P2 wait(S2),wait(S1) => Deadlock moeglich.

Producer / Consumer

mutex=1, empty=N, full=0

Prod: wait(empty), wait(mutex), insert, signal(mutex), signal(full)

Cons: wait(full), wait(mutex), remove, signal(mutex), signal(empty)

9. Deadlocks

Deadlock: zyklische Wartebeziehung, keiner kann die benoetigte Ressource freigeben.

  • Mutual Exclusion
  • Hold and Wait
  • No Preemption
  • Circular Wait
R -> PRessource R ist P zugeordnet.
P -> RP wartet auf Ressource R.
P -> QP wartet auf Ressource, die Q haelt.
Eine Instanz je Ressource: Zyklus => Deadlock. Mehrere Instanzen: Zyklus nur Warnsignal.

Mehrinstanz-Pruefung

  1. Freie Instanzen zaehlen.
  2. Prozess finden, dessen Request erfuellbar ist oder der nicht wartet.
  3. Als fertig markieren, Ressourcen freigeben.
  4. Wiederholen. Bleibt ein wartender Rest => Deadlock.

10. Dateisysteme

belegt = ceil(Dateigroesse / Einheit) * Einheit

Einheit = Block oder Cluster. Immer auf ganze Einheit runden.

FAT / Cluster

FAT-Eintraege = 2^b
max Partition = Eintraege * Block/Cluster
FAT-Groesse = Eintraege * Bytes/Eintrag

16-bit FAT, 512 B: 2^16*512=32 MiB.

Cluster i=3: 8*512=4096 B, max 256 MiB.

652 B Datei: ohne Cluster 1024 B, mit Cluster 4096 B.

Belegungsarten

zusammenh.schnell; aber Fragmentierung/Wachstum.
verkettetkein zusammenh. Platz; Random schlecht, Zeigerplatz.
FATKette in Tabelle; FAT gross, Clusterverlust.
I-NodeIndex mit direkten/indirekten Adressen.

11. I-Node

ptr = Blockgroesse / Adressgroesse
Daten = direkt + einfach*ptr + doppelt*ptr^2 + dreifach*ptr^3
MaxBytes = Datenbloecke * Blockgroesse
Nutzdateigroesse: nur Datenbloecke. Plattenplatz: Daten + I-Node + Zeigerbloecke.

Beispiel: 8 direkte + 4 einfach indirekte, 128 Adr./Block, Block 512:

Daten = 8 + 4*128 = 520

Max = 520*512 = 266240 B = 260 KiB

13. Fallenliste

  • EDF: absolute Deadline rechnen.
  • RMS: nach Periode, nicht nach C.
  • RMS-Grenze nur hinreichend.
  • Ready != Blocked.
  • Semaphor initial 0/1/N unterscheiden.
  • Mehrinstanzen: Zyklus allein reicht nicht.
  • Paging: Offsetbits aus Pagegroesse.
  • Array: Startoffset beachten.
  • Cache: Miss laedt ganzen Block.
  • Liste: Zeigerplatz abziehen.
  • FAT/Cluster: aufrunden.
  • I-Node: Nutzgroesse vs. Plattenplatz.
  • Theorie: Definition + Eigenschaft + Beispiel.
