Scheduling / Echtzeit

Allgemeines Vorgehen Zeitdiagramm

  1. Tasks notieren: Ankunft A, Rechenzeit C, Deadline D.
  2. Bei A+x: absolute Deadline = A+x, nicht nur x.
  3. Nur an Ereignissen entscheiden: Ankunft, CPU-Ende, Quantum-Ende, Deadline.
  4. Ready Queue: nur lauffaehige Jobs, Blocked ist nicht ready.
  5. Verdrängung nur, wenn Regel neuen Job bevorzugt.
  6. Kontrolle: Summe der Balken je Task = Rechenzeit.
VerfahrenRegelFalle

EDF: Job mit kleinster absoluter Deadline gewinnt. Kommt neuer Job mit frueherer Deadline, verdrängt er.
LLF: Laxity = Deadline - Restzeit - aktuelle Zeit. Kleinste Reserve gewinnt.

RMS Formeln

Priorität: kleineres P_i bzw. hoehere Rate 1/P_i = hoehere Priorität.
Auslastung: U = sum(C_i/P_i). Wenn U > 1: auf einem Prozessor nicht machbar.
Hinreichende Garantie: U <= n(2^(1/n)-1). Werte: n=2: 0,828; n=3: 0,779; n→∞: ln2=0,693.
Wichtig: Grenze ist hinreichend, nicht notwendig. Oberhalb der Grenze nicht automatisch falsch; genauer zeichnen/Antwortzeit pruefen.

Antwortzeitprüfung feste Prioritäten

R_i = C_i + sum_{j hoeher} ceil(R_i / P_j) * C_j

Iteration: mit R=C_i starten, bis stabil. Deadline ok, wenn R_i <= D_i bzw. meist P_i.

Beispiel T1/T2: T1 C=7,P=13, T2 C=6,P=18. R1=7. R2=6+ceil(R2/13)*7 → 13. Beide ok.

Paging / Adressrechnung

Pagegröße: pageSize = 2^k → Offset-Bits = k.
Seitennummer-Bits: Adressbits - Offsetbits.
virtuelle Seiten: 2^(Adressbits-k).
Frames: physischer Speicher / pageSize.
Seitentabelle: Einträge * Bytes_pro_Eintrag.
4 KiB = 2^12 → 12 Offsetbits = letzte 3 Hex-Ziffern. 0x4B30A568 → Page 0x4B30A, Offset 0x568, Bereich 0x4B30A000..FFF.

Physische Adresse

  1. Virtuelle Adresse in Page + Offset splitten.
  2. Seitentabelle: Page → Frame.
  3. Physische Adresse = Frame-Bits + unveraendertes Offset.

Array über Pages

Pages = floor((StartOffset + Laenge - 1) / PageSize) + 1

Minimum ceil(L/PageSize), Maximum bei unguenstigem Startoffset.

TLB / Seitentabellen / Page Fault

TLB-Zeit

EAT = hitRate*T_hit + missRate*T_miss
Mit TLB 10 ns, RAM 100 ns, Hit 90%: Hit=110, Miss=210, EAT=0,9*110+0,1*210=120 ns.

Mehrstufige Seitentabelle

Nur angelegte Tabellen zählen. Beispiel 32 Bit, 4 KiB, 10/10/12, Eintrag 4 B: eine Tabelle = 2^10*4=4096 B. 1 L1 + 3 L2 = 16 KiB.

Page-Fault-Ablauf

  1. Present-Bit 0 → Trap in Kernel.
  2. Prozess blockieren.
  3. Freien Frame suchen oder Opfer wählen.
  4. Dirty Opferseite zurückschreiben.
  5. Benötigte Seite laden.
  6. Seitentabelle/TLB aktualisieren.
  7. Instruktion später erneut ausführen.
Thrashing: Working Sets passen nicht in Frames → sehr viele Page Faults. Gegenmaßnahme: Multiprogramminggrad senken / mehr Frames.

Cache-Rechnung

Block-Misses sequentiell: ceil(Bytes / Blockgröße), falls Start blockbündig.
Hit-Rate: Hits / Zugriffe. Miss-Rate = 1 - Hit-Rate.
Mittlere Zeit: HitRate*T_cache + MissRate*T_miss oder laut Aufgabe inkl. Cachezugriff.
200 Byte, Block 64, Cold Cache: Blöcke 0-63,64-127,128-191,192-255 → 4 Misses, 196 Hits, Hit-Rate 98%.

Page Replacement

Tabellen-Schema

  1. Referenzstring links nach rechts.
  2. Hit: Zustand bleibt, aber LRU/Use-Bit aktualisieren.
  3. Fault mit freiem Frame: laden.
  4. Fault ohne freien Frame: Opfer nach Verfahren.
  5. Nach jedem Schritt komplette Frame-Tabelle schreiben.

Synchronisation

Race Condition / kritischer Abschnitt

Race Condition: Ergebnis hängt vom zeitlichen Ablauf nebenläufiger Zugriffe ab. Kritischer Abschnitt: Zugriff auf gemeinsame Ressource, exklusiv schützen.

MechanismusKernFalle
Interrupts sperrenkeine Unterbrechung auf 1 CPUMultiprozessor ungeeignet
Test-and-Setatomar prüfen+setzenBusy Waiting
Mutexexklusiver Abschnittalle Beteiligten müssen ihn nutzen
SemaphoreZähler mit wait/signalInitialwert entscheidet Bedeutung
Monitorgekapselte Daten + exklusive ProzedurenCondition wait gibt Monitor frei

Semaphore

Mutex: initial 1; vor Abschnitt wait/P, danach signal/V.
Reihenfolge: Semaphor initial 0; Prozess wartet, anderer signalisiert Ereignis.
Ressourcenpool: initial = Anzahl gleichartiger Ressourcen.
Semaphor schützt nicht automatisch. Nur korrekt platzierte wait/signal schützen.

Semaphor-Code prüfen

  1. Initialwerte notieren.
  2. Bei jedem wait(S): danach hält Prozess S bzw. eine Ressource?
  3. Bei jedem signal(S): wird S wieder freigegeben?
  4. Kritischer Abschnitt: nutzen alle Prozesse denselben Mutex?
  5. Deadlock: P1 hält S1 wartet S2, P2 hält S2 wartet S1?
  6. Mehrere Locks immer gleiche globale Reihenfolge?
Klassiker:
P1: wait(S1); wait(S2); ...
P2: wait(S2); wait(S1); ...
⇒ Deadlock möglich. Fix: gleiche Reihenfolge.

Producer/Consumer

mutex=1, empty=N, full=0

Producer: wait(empty); wait(mutex); insert; signal(mutex); signal(full)

Consumer: wait(full); wait(mutex); remove; signal(mutex); signal(empty)

Einmalige Barriere

mutex=1, turnstile=0, count=0

Letzter Thread öffnet turnstile. Wiederverwendbare Barriere braucht mehr Sorgfalt, meist zwei Turnstiles.

Deadlocks

Definition

Zyklische Wartebeziehung: Jeder wartet auf Ressource/Ereignis, das nur ein anderer aus der Menge freigeben kann.

Vier Coffman-Bedingungen

  • Mutual Exclusion: Ressource exklusiv.
  • Hold and Wait: hält etwas und wartet weiter.
  • No Preemption: Ressource nicht zwangsweise entziehbar.
  • Circular Wait: zyklische Wartekette.

Ressourcengraph

KanteBedeutung
R → PRessource ist P zugeordnet / P hält R.
P → RP wartet auf Ressource R.
P → Q im Wait-forP wartet auf Ressource, die Q hält.
Eine Instanz pro Ressource: Zyklus ⇒ Deadlock. Mehrere Instanzen: Zyklus nur Warnsignal; prüfen, ob freier/nicht wartender Prozess Ressourcen freigeben kann.

Mehrinstanz-Reduktion

  1. Freie Instanzen zählen.
  2. Prozess finden, dessen Request erfüllbar ist oder der nicht wartet.
  3. Als fertig markieren, seine Ressourcen freigeben.
  4. Wiederholen. Bleiben wartende Prozesse übrig: Deadlock.

Dateisysteme

Einheiten / Rundung

belegter Platz: ceil(Dateigröße / Einheit) * Einheit

Einheit = Block oder Cluster. Immer ganze Einheiten belegen.

FAT / Cluster

Einträge: 2^b bei b Bit.
max Partition: Einträge * Blockgröße oder Einträge * Clustergröße.
FAT-Größe: Einträge * Bytes_pro_Eintrag.
16-bit FAT, 512 B: 2^16*512 = 32 MiB. Cluster i=3: 8*512=4096 B; max = 256 MiB. 652 B-Datei: ohne Cluster 1024 B, mit Cluster 4096 B.

Maximale Nutzdateigröße

Datenblöcke = direkt + einfach + doppelt + dreifach
Adressen_pro_Zeigerblock = Blockgröße / Adressgröße
MaxBytes = Datenblöcke * Blockgröße
EbeneDatenblöcke
direktAnzahl direkte Einträge
einfach indirektptr

8 direkte + 4 einfach indirekte, 128 Adressen pro Zeigerblock, Block 512: Datenblöcke = 8+4*128=520; Max = 520*512 = 266240 B = 260 KiB.

Plattenverbrauch

Gesamt = Datenblöcke + I-Node-Block + benötigte Zeigerblöcke

Nur mitzählen, wenn nach Platz auf Platte gefragt ist. Bei Nutzdateigröße zählen Zeigerblöcke nicht als Nutzdaten.

Kurzantwort-Bausteine

Prozess: Programm in Ausführung mit eigenem Adressraum, Ressourcen und PCB.
Thread: Ausführungsfaden; teilt Adressraum/Ressourcen des Prozesses, hat eigenen Stack/Register.
Context Switch: Zustand des alten Prozesses sichern, Zustand des neuen laden, Dispatcher übergibt CPU.
Interrupt: asynchrone Unterbrechung; ISR behandelt Ereignis, danach Fortsetzung.
DMA: Controller überträgt Daten Gerät↔RAM; CPU initialisiert, Abschluss per Interrupt.
Monitor: kapselt Daten/Prozeduren; nur ein Thread gleichzeitig aktiv.
Semaphore: expliziter Zähler mit atomarem wait/signal; Initialwert wichtig.
TLB: Cache für virtuelle→physische Adressübersetzungen; spart Seitentabellenzugriffe.
Thrashing: Working Sets passen nicht; System verbringt Zeit mit Paging statt Nutzarbeit.
IPC: Pipes, Message Queues, Shared Memory, Sockets, Signale; Shared Memory braucht Synchronisation.

Fallenliste

  • EDF: absolute Deadline, nicht relative Zahl.
  • RMS: nach Periode sortieren, nicht nach Rechenzeit.
  • RMS-Grenze: hinreichend, nicht notwendig.
  • Ready != Blocked.
  • Verdrängung != freiwilliges Blockieren.
  • Semaphor initial 0 oft Reihenfolge, initial 1 oft Mutex.
  • Bei Deadlock braucht man Zyklus bzw. Mehrinstanzprüfung, nicht nur Warten.
  • Mehrere Instanzen: Zyklus allein beweist keinen Deadlock.
  • Paging: Offsetbits aus Pagegröße, nicht aus Adressbreite raten.
  • Array über Pages: Startoffset beachten.
  • Cache: Ein Miss lädt ganzen Block, nicht nur ein Byte.
  • Verkettete Liste: Zeigerplatz vom Datenblock abziehen.
  • FAT/Cluster: belegten Platz immer aufrunden.
  • I-Node: Nutzdateigröße vs. Plattenplatz unterscheiden.
  • Theorie: Definition + 1 Eigenschaft + Abgrenzung/Beispiel.
